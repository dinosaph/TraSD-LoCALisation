1. Abstract

The following article presents you the development of a service-based web application in which users receive defined traffic sign detection data by providing video data (through webcam/video file upload). The application was developed during InfoIasi College Web Application Development Course, in autumn-winter 2021 and it uses Tensorflow.js (for the classification), html, css, js (frontend), node.js (backend), MongoDB (database) and traffic related ontology technologies.

2. Introduction

The project's starting idea was that we need a service-based web application through which users could receive traffic sign data output by providing video input or by collectively collected street/traffic data (such as in Waze app).

S@fe web application is about providing useful live or past traffic data. It sees, it detects traffic signs and it tells you useful details that it defines with the help of the detected data. Our chosen development path involves the usage of video input through live webcam streaming or video file upload, these being turned into frames and further studied for traffic sign detection. The project follows a minimalist approach and design, therefore being able to attract potential S@fers in need of the provided output data. You need to be S@fe, to learn which travel path is S@fer, to study the traffic in the area you aren't sure about? You might need our S@fe hand.

3. Architecture

S@fe application is proposed to be developed in the following 4 modules:

  • Web Interface - Minimalist and friendly design
  • Server - Main server dealing with basic operations (e.g.: GET /Home)
  • Scanner - Server that deals with the traffic sign scan and communicates with the Classifier
  • Classifier - Classifying module that deals with the video input data => provides output data

    • 3.1 Web Interface

    UI module with the following functionality:
    > The user enters website and walks through different web pages - Home, About, Scan, Contact
    > The user wants to receive helpful traffic data so he/she goes to the SCAN page and follows the directions.
    > The user uploads video file or chooses to stream his/her webcam, therefore output data will be shown next to the video. The output data is about the traffic signs detected on the video + defined with additional helpful traffic ontology data. The user can also request a generated report related to the collected traffic sign data.

    3.1.1 Page - HOME

    Welcoming web page that briefly explains/provides overview for the app.

    3.1.2 Page - ABOUT

    Informative web page related to the project and the team.

    3.1.3 Page - SCAN

    Web page related to the traffic signs scan where users provide video data for further backend operations and receive traffic scan data.

  • User clicks UPLOAD to provide video file data
  • User clicks WEBCAM to provide video data through webcam
  • Video data turns to frames and is used in the backend for live detection
  • User gets live traffic sign detection + definition data
  • User can pause/stop/play the video for study purposes
  • User can request the collected traffic scan report

    • 3.1.4 Page - CONTACT

    Web page for app users -> project team interaction.

    3.2 Server

    Backend component dealing with all main operations (not related to the traffic scan)

  • Purpose: Provides website/web pages to users
  • Used technologies: node.js

    • 3.3 Scanner

    Backend component dealing with all traffic scan operations

    Purpose:
  • provides traffic scan defined data
  • provides traffic scan report
  • provides interactive traffic study

    • Technologies:
  • node.js
  • react.js

    • 3.4 Classifier

    Backend component dealing with all traffic sign classifications operations

    Purpose:
  • provides traffic sign classified data

    • Technologies:
  • Tensorflow.js
  • romanian traffic sign training data

    • 4. Data Modelling

    S@fe application will be storing needed information in a database that is popular, tested, documented, easy to use and lightweight, MongoDB

    5. Development Conclusions

    The team proposed the idea to be completely implemented in about 3-4 weeks (maximum) and maximum excitement.
